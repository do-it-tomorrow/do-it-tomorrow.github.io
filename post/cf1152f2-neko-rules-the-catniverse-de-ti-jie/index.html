CF1152F2 Neko Rules the Catniverse 的题解

· 2024-11-08 · # 计数问题 # 动态规划 # Codeforces # 题解

按照一般得思路，我们应该记录那些元素被选择了。可是 nnn 太大了，考虑转化一下。

mmm 很小只有 444，这启发我们从 111 开始到 nnn 一个一个考虑，思考一下填入需要满足什么要求：

具体的，我们考虑在一个序列中插入一些数，使这个序列一直满足题目的要求。

首先第 1,21,21,2 个要求是肯定满足得，因为我们就是在枚举要填入什么元素，所以肯定不会出现填入的数大于 nnn 或者有重复的情况。

因为插入的顺序是递增的，所以一定有 ai<ai−1a_i<a_{i-1}ai​<ai−1​，也就一定满足 ai≤ai+1+ma_i\le a_{i+1}+mai​≤ai+1​+m。

所以新填入的数与后面的关系一定是满足题目要求的，只需要考虑与前面的关系就行了。

注意这里有一个细节就是第 111 个位置可以随便填，因为它根本就没有前面的元素。

考虑会影响填入 xxx 的有什么，显然 xxx 只有插入到 [x−m,x)[x-m,x)[x−m,x) 这些数后面才行，那么暴力的考虑，直接状压记录这 mmm 个位置都填的什么。

在考虑了上面之后，状态就很好想出来了。

设 fi,j,Sf_{i,j,S}fi,j,S​ 表示现在考虑到了填 iii，已经填了 jjj 个数，[x−i,i)[x-i,i)[x−i,i) 这个区间内都用了 SSS 的数，那么有转移方程：

fi+1,j+1,(2S+1) mod (2m)←fi,j,S×(popcount⁡(S)+1)f_{i+1,j+1,(2S+1)\bmod(2^m)}\gets f_{i,j,S}\times (\operatorname{popcount}(S)+1) fi+1,j+1,(2S+1)mod(2m)​←fi,j,S​×(popcount(S)+1)

fi+1,j,(2S) mod (2m)←fi,j,Sf_{i+1,j,(2S)\bmod (2^m)}\gets f_{i,j,S} fi+1,j,(2S)mod(2m)​←fi,j,S​

这样的时间复杂度为 O(n×k×2m)O(n\times k\times 2^m)O(n×k×2m) 的，可以通过 F1。

发现第 2,32,32,3 维其实非常小，上限只有 192192192，考虑使用矩阵快速幂优化。

因为如果处理二维矩阵很麻烦，考虑先把矩阵压缩成一维的，设原来的位置 (x,y)(x,y)(x,y) 为 p(x,y)p(x,y)p(x,y)。

那么如果现在要将 (x,0)×z(x,0)\times z(x,0)×z 转移到 (y,0)(y,0)(y,0)，那么只需要将 (x,y)(x,y)(x,y) 设置为 zzz 就行了。

时间复杂度为 O(log⁡n×(k×2m)3)O(\log n\times (k\times 2^m)^3)O(logn×(k×2m)3)，可以通过 F2。
